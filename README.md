# Deepfake Audio Detection System
### Vishing (Voice Phishing) Defense

A machine learning system that detects AI-generated (deepfake) voices in audio recordings. Upload any voice recording and get an instant prediction with a confidence score.

---

## What It Does

- Analyses audio files to determine if a voice is **real** or **AI-generated**
- Returns a confidence score (0–100%)
- Displays a waveform visualization of the audio
- Warns about potential vishing (voice phishing) threats

---

## Project Files

- `Deepfake_Audio_Detection_System_for_Voice_Phishing_Defense.ipynb` — Colab notebook for training the model
- `streamlit_app.py` — Web app for running predictions
- `voice_model.pkl` — Saved model file (generated after training)

## Setup

### Step 1 — Train the Model (Google Colab)

1. Open the notebook in Google Colab
2. Run Steps 1–9 in order to train and save the model
3. Download `voice_model.pkl` to your project folder

### Step 2 — Run the Web App (Local) (not needed optional)

Install dependencies:

```bash

```

Run the app:

```bash

```

Open the URL shown in your terminal (usually `http://localhost:8501`).

### Alternative — Run in Colab via ngrok

Follow Step 11 in the notebook to run the web app directly inside Colab using an ngrok tunnel.

## Dataset Structure

Organise your training audio files like this:

```
dataset/
  real/
    voice1.wav
    voice2.wav
  fake/
    deepfake1.wav
    deepfake2.wav
```

Accepted formats: `.wav`, `.mp3`, `.flac`, `.ogg`

---

## How It Works

1. Audio is loaded and trimmed to 5 seconds at 22050 Hz
2. Features are extracted using librosa — chroma, spectral contrast, zero crossing rate, and RMS energy
3. Features are normalised using a standard scaler
4. A Random Forest classifier (200 trees) predicts real vs. fake
5. The confidence score is the model's probability for the predicted class

## What is Vishing?

Voice phishing (vishing) is a social engineering attack where criminals use phone calls — increasingly with AI-cloned voices — to trick victims into revealing sensitive information such as passwords, banking details, or personal data. This system helps identify whether a voice recording was generated by an AI.
